7. prednáška

Zakódujte danú postupnosť znakov pomocou Huffmannovho kódu – vytvorte a nakreslite kódový strom (to je všetko)

  • Priklad:
  • Vstup (frekvencie znakov, často uvádzaný v literatúre):
    • a : 5
    • b : 9
    • c : 12
    • d : 13
    • e : 16
    • f : 45
  1. Postup zostavenia Huffmanovho stromu (zjednodušené kroky)
    1. Zoradíme uzly podľa frekvencie (od najmenšej):
      a(5), b(9), c(12), d(13), e(16), f(45)
    2. Zoberieme dva najmenšie uzly a vytvoríme ich rodiča s frekvenciou súčtu:
      • zmergovať a(5) + b(9) → uzol ab(14)
      • nový zoznam (zoradený):
        c(12), d(13), ab(14), e(16), f(45)
    3. Zoberieme dve najmenšie: c(12) + d(13) → cd(25).
      • nový zoznam:
        ab(14), e(16), cd(25), f(45)
    4. Zoberieme dve najmenšie: ab(14) + e(16) → abe(30).
      • nový zoznam:
        cd(25), abe(30), f(45)
    5. Zoberieme dve najmenšie: cd(25) + abe(30) → cdabe(55).
      • nový zoznam:
        f(45), cdabe(55)
    6. Zoberieme posledné dve: f(45) + cdabe(55) → koreň(100).
  2. Priradenie bitov

    Štandardný princíp: pri každom rozvetvení priradíme jednej vetve bit 0 a druhej vetve bit 1 (poradie 0/1 je konvencia — výsledné kódy sú platné pri ľubovoľnej konzistentnej voľbe). Použijem konvenciu: ľavá vetva = 0, pravá vetva = 1.

  3. Výsledný strom (ASCII diagram)
  4. Kódy pre jednotlivé znaky (čítaním od korena: ľavá = 0, pravá = 1)
    • f: 0 (f je priamo v ľavej vetve koreňa → krátky kód, lebo má najväčšiu frekvenciu)

      • Pre pravú vetvu (prefix 1):

    • c : 100
    • d : 101
    • a : 1100
    • b : 1101
    • e : 111
      • (tieto kódy sú presne známe z poučného príkladu; dĺžky zodpovedajú frekvenciám — častejší f má 1 bit, najriedkejší a má 4 bity)
  5. Príklad zakódovania reťazca

    6. Ak chceme zakódovať napr. reťazec "abcdef", použijeme mapovanie:

    • a → 1100
    • b → 1101
    • c → 100
    • d → 101
    • e → 111
    • f → 0

    Kódovaný bitový reťazec:

    1100 1101 100 101 111 0 → spojene: 110011011001011110

  6. Poznámky
    • Huffmanov kód je prefixový (žiadny kód nie je prefixom iného) — dekódovanie je jednoznačné.
    • Ak by sme pri zlučovaní menili poradie priradenia 0/1, kódy by sa líšili, ale dĺžky zostávajú rovnaké (efektívnosť zachovaná).
